# SSL+PU学习实施方案 - 快速开始指南

## 🎯 项目目标
基于CPSC2021 ECG数据集，实现SSL+NNPU+xResNet50的房颤检测模型，预期性能：
- 准确率 > 85%
- F1分数 > 80%
- ROC-AUC > 0.85

## ⚙️ 关键配置参数

### 数据参数
- **输入维度**: 单导联ECG，长度4000
- **训练集**: 20000个样本
  - AF样本（正例）: 500个 (标签=1)
  - Non-AF样本（负例）: 500个 (标签=0)
  - 无标签样本: 19000个 (标签=-1)
- **先验概率**: π = 0.0256

### 训练参数
- **批次大小**: 32
- **SSL预训练**: 100轮，学习率1e-3
- **NNPU微调**: 50轮，学习率1e-4
- **PU损失权重**: 5.0
- **SSL温度参数**: 0.1

## 📊 训练流程

### 阶段1：SSL预训练（无监督）
- **数据**: 19000个无标签样本
- **目标**: 学习ECG信号的通用特征表示
- **方法**: SimCLR对比学习
- **输出**: 预训练的xResNet50编码器

### 阶段2：NNPU微调（半监督）
- **数据**: 500个AF样本 + 19000个无标签样本
- **目标**: 学习AF检测分类器
- **方法**: 非负PU学习（Imbalanced nnPU）
- **输出**: 最终的AF检测模型

## 🔍 监控指标

训练过程中会监控以下指标：

### 基础指标
- **损失值**: SSL损失、PU损失
- **准确率**: 整体分类准确率
- **精确率**: AF样本的精确识别率
- **召回率**: AF样本的检出率
- **F1分数**: 精确率和召回率的调和平均

### 高级指标
- **ROC-AUC**: 接收者操作特征曲线下面积
- **PR-AUC**: 精确率-召回率曲线下面积
- **混淆矩阵**: 详细的分类结果分布

### Prior-corrected指标
- **修正准确率**: 考虑真实先验概率的准确率
- **修正AUC**: 考虑真实先验概率的AUC

## 🔧 常见问题

### Q1: OpenMP运行时冲突
```
OMP: Error #15: Initializing libiomp5md.dll, but found libiomp5md.dll already initialized.
```
**解决方案**: 设置环境变量
```bash
set KMP_DUPLICATE_LIB_OK=TRUE
```

### Q2: GPU内存不足
**解决方案**: 减小批次大小
```python
BATCH_SIZE = 16  # 或更小
```

### Q3: 训练速度慢
**建议**:
- 使用GPU训练
- 增加num_workers数量
- 使用混合精度训练

## 📈 性能优化建议

### 1. 模型架构优化
- 尝试ResNet101或更深的网络

- 更多样的ECG增强方法
- 基于生理知识的增强
- 对抗训练
